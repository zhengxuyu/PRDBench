# OA System Test Plan

This directory contains the complete OA System test plan, generated based on the functional evaluation metrics defined in `metric.json`.

## File Structure

```
evaluation/
├── README.md                    # This documentation file
├── metric.json                  # Original functional evaluation metrics
├── detailed_test_plan.json      # Detailed test plan
├── tests/                       # Automated test scripts
│   ├── __init__.py
│   ├── test_system_init.py      # System initialization tests
│   ├── test_auth_service.py     # Authentication service tests
│   └── test_workflow_service.py # Workflow service tests
├── *.in                        # Test input files
└── expected_*.csv              # Expected output files
```

## Test Type Description

### 1. Shell Interaction Tests (Command-Line Interaction Tests)
These tests simulate real user interaction with command-line programs to verify the program's user interface and interaction flow.

**Execution Method:**
```bash
# Example: Test user login
python src/main.py login admin < evaluation/admin_login.in
```

### 2. Unit Tests
Automated tests using the pytest framework to verify core business logic and service functionality.

**Execution Method:**
```bash
# Run all unit tests
pytest evaluation/tests/

# Run specific tests
pytest evaluation/tests/test_system_init.py::test_database_initialization
pytest evaluation/tests/test_auth_service.py::test_user_authentication
pytest evaluation/tests/test_workflow_service.py::test_workflow_operations
```

### 3. File Comparison Tests
Verify that the output files generated by the program meet the expected format and content.

**Execution Method:**
```bash
# Generate output files
python src/main.py < evaluation/data_export.in

# Compare generated files with expected files
diff export.csv evaluation/expected_export.csv
```

## Test Execution Steps

### Prerequisites
1. Ensure Python environment has the necessary dependencies installed:
   ```bash
   pip install pytest sqlalchemy rich click
   ```

2. Initialize system database:
   ```bash
   python src/main.py
   # Select "2. Initialize System"
   ```

### Execute Tests

#### 1. Run Unit Tests
```bash
# Navigate to project root directory
cd /path/to/OA

# Run all unit tests
pytest evaluation/tests/ -v

# Run specific test modules
pytest evaluation/tests/test_system_init.py -v
pytest evaluation/tests/test_auth_service.py -v
pytest evaluation/tests/test_workflow_service.py -v
```

#### 2. Run Command-Line Interaction Tests
```bash
# Test system help information
python src/main.py --help

# Test user login
python src/main.py login admin < evaluation/admin_login.in

# Test user list
python src/main.py user list

# Test workflow templates
python src/main.py workflow templates

# Test user creation
python src/main.py user create < evaluation/user_create.in
```

#### 3. Run File Comparison Tests
```bash
# Execute data export function
python src/main.py < evaluation/data_export.in

# Check generated files
ls -la export.csv

# Compare file content (if export functionality is supported)
diff export.csv evaluation/expected_export.csv
```

## Expected Results

The expected results for each test case are detailed in `detailed_test_plan.json`. When executing tests, verify actual output against expected results.

### Success Criteria
- **Shell Interaction Tests**: Program executes normally, output contains expected information, no crashes
- **Unit Tests**: All assertions pass, test cases execute successfully
- **File Comparison Tests**: Generated files match expected file content

### Failure Handling
If tests fail, please check:
1. Whether the system is initialized correctly
2. Whether database connection is normal
3. Whether input data format is correct
4. Whether program logic meets expectations

## Notes

1. **Database Isolation**: Unit tests use independent test databases and do not affect the main database
2. **Test Order**: Some tests may depend on system initialization; it's recommended to run system initialization first
3. **Environment Cleanup**: Test databases are automatically cleaned up after testing
4. **Permission Requirements**: Some tests require administrator privileges; ensure you use the correct user account

## Extended Testing

To add new test cases:

1. **Shell Interaction**: Add new test items in `detailed_test_plan.json` and create corresponding `.in` input files
2. **Unit Tests**: Create new test files in the `tests/` directory following the `test_*.py` naming convention
3. **File Comparison**: Create expected output files and specify file paths in the test plan

## Technical Support

If you encounter test execution problems, please check:
- Python version compatibility (Python 3.8+ recommended)
- Complete dependency package installation
- File paths and permission settings
- Database configuration correctness
