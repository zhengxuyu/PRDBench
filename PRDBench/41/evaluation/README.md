# 3-MSA Multi-Sequence Alignment Algorithm Optimization System - Test Plan

## Overview

This test plan provides a complete automated testing framework for the 3-MSA Multi-Sequence Alignment Algorithm Optimization System, containing 29 test metrics that cover all major functionalities of the system.

## File Structure


## Test Types

### 1. Shell Interaction Tests (shell_interaction)
- **Purpose**: Test user interaction with command-line program
- **Quantity**: 24 tests
- **Examples**: Main menu display, input validation, algorithm selection, etc.

### 2. Unit Tests (unit_test)
- **Purpose**: Test implementation of core algorithms and optimization techniques
- **Quantity**: 3 tests
- **Examples**: Dynamic programming algorithm, A* search algorithm, genetic algorithm

### 3. File Comparison Tests (file_comparison)
- **Purpose**: Verify that output files generated by the program are correct
- **Quantity**: 2 tests
- **Examples**: Result saving function, configuration information completeness

## Quick Start

### 1. Environment Preparation

Ensure necessary dependencies are installed:
```bash
pip install pytest
```

### 2. Run All Tests

```bash
# Execute in project root directory
python evaluation/run_tests.py
```

### 3. Run Specific Type of Tests

```bash
# Run unit tests
pytest evaluation/tests/ -v

# Run specific unit test
pytest evaluation/tests/test_dp_algorithm.py::test_dp_execution -v

# Run fixed version genetic algorithm tests
pytest evaluation/tests/test_ga_algorithm_fixed.py -v
```

### 4. Manually Run Shell Interaction Tests

```bash
# Test program startup
python src/main.py < evaluation/input_files/main_menu_test.in

# Test invalid input validation
python src/main.py < evaluation/input_files/invalid_input_test.in
```

## Custom Testing

### Adding New Test Cases

1. **Shell Interaction Tests**:
   - Create new `.in` files in `input_files/` directory
   - Add corresponding test configuration in `detailed_test_plan.json`

2. **Unit Tests**:
   - Create new test files in `tests/` directory
   - Write test functions using pytest framework

3. **File Comparison Tests**:
   - Create expected output files in `expected_output/` directory
   - Configure `expected_output_files` field in test plan

### Modifying Test Parameters

Edit `detailed_test_plan.json` file to modify test configuration:
- `test_command`: Test execution command
- `test_input`: Input file path
- `expected_output`: Expected output description

## Troubleshooting

### Common Issues

1. **Test Timeout**
   - Check if algorithm implementation has infinite loops
   - Adjust timeout settings in `run_tests.py`

2. **File Path Errors**
   - Ensure tests are executed in project root directory
   - Check if input files and expected output files exist

3. **Genetic Algorithm Division by Zero Error**
   - Use test cases in `test_ga_algorithm_fixed.py`
   - Refer to fix recommendations to improve algorithm implementation

### Debugging Tips

1. **View Detailed Output**:
   ```bash
   python evaluation/run_tests.py 2>&1 | tee test_output.log
   ```

2. **Run Failed Tests Individually**:
   ```bash
   python src/main.py < evaluation/input_files/specific_test.in
   ```

3. **Check Test Report**:
   View generated `evaluation/test_execution_report.json` file
