# Test Dependency Generation Prompt

## Task Description
Based on the test plan in `detailed_test_plan.json`, analyze the dependencies between test points and generate the `test_dependencies.json` file.

## Analysis Principles

### 1. Data Dependency Analysis
- **Enterprise Information Entry** test points (2.1.x) serve as the foundational data source for other functionalities
- **Financing Diagnosis Analysis** test points (2.2.x) require enterprise information as input
- **Report Generation** test points (2.3.x) require diagnosis results as input

### 2. Functional Dependency Analysis
- Review the "prerequisite checks" and "preparation" sections in test descriptions
- Identify test points that require "completed", "ensure exists", and similar expressions
- Analyze logical sequence requirements in test flows

### 3. Historical Data Dependency Analysis
- **Historical Report Management** (2.3.5a) requires multiple report generations to provide test data
- **Report Comparison** (2.3.5b) requires at least 2 historical reports
- **Trend Analysis** type functionalities require time series data

### 4. Dependency Relationship Types
- **Direct Dependency**: Functionality A directly needs the output of functionality B
- **Data Dependency**: Test A needs data generated by test B
- **Prerequisite Dependency**: Test A needs functionality validated by test B to work properly
- **Batch Dependency**: Test A needs multiple test points to complete to obtain sufficient data

## Output Format

```json
[
    {
        "description": "Brief explanation of the dependency relationship",
        "source": "Prerequisite test point ID or array of IDs",
        "target": "Target test point ID"
    }
]
```

## Example Analysis

Based on descriptions in the test plan:
- If a test point description mentions "after completing financing diagnosis", it depends on 2.2.x series test points
- If a test point description mentions "generate 2-3 diagnosis reports at different times", it depends on report generation related test points
- If a test point description mentions "ensure at least 2 historical reports from different periods exist", it depends on historical report archiving test points

## Analysis Steps

1. **Read Test Plan**: Carefully read the description field of each test point
2. **Identify Keywords**: Look for key expressions like "depends on", "requires", "after completing...", "ensure exists", etc.
3. **Analyze Test Types**: Determine functional classification of test points based on metric names
4. **Establish Dependency Chains**: Build dependency chains from basic functionalities to advanced functionalities
5. **Verify Completeness**: Ensure all dependency relationships have clear source and target

Please analyze `detailed_test_plan.json` according to the above principles and generate a complete `test_dependencies.json` file.
