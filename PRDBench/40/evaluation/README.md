# MinNE Network Simulation System Test Plan

This directory contains a complete test plan for evaluating the functional completeness of the MinNE Layered Network Simulation and Protocol Stack Implementation Platform.

## File Structure

### Core Files
- `metric.json` - Functional evaluation metric definitions
- `detailed_test_plan.json` - Detailed test plan containing 30 test points
- `README.md` - This documentation file

### Input Files
- `port_query.in` - Port query test input
- `text_transmission_test.in` - Text transmission test input
- `utf8_test_text.txt` - UTF-8 encoding test text
- `image_transmission_test.in` - Image transmission test input
- `image_save_test.in` - Image save test input
- `image_naming_test.in` - Image naming test input
- `console_test.in` - Console function test input
- `large_file_test.in` - Large file transmission test input
- `test_image.png` - Test image file
- `large_test_file.bin` - Large file test data
- `crc_test_data.txt` - CRC verification test data

### Automated Test Scripts (tests/)
- `test_frame_structure.py` - Frame structure field completeness test
- `test_frame_classes.py` - FrameBuilder and FrameParser class tests
- `test_crc16.py` - CRC-16 algorithm test
- `test_switch_table.py` - Switch address table test
- `test_switch_learning.py` - Switch address learning test
- `test_switch_lifetime.py` - Switch lifetime management test
- `test_router_table.py` - Routing table structure test
- `test_dijkstra.py` - Dijkstra algorithm test
- `test_buffer_config.py` - Buffer configuration test
- `test_flow_control.py` - Flow control mechanism test
- `test_bit_stuffing.py` - Bit stuffing algorithm test
- `test_arq.py` - Stop-and-Wait ARQ protocol test
- `test_switch_forwarding.py` - Switch forwarding function test

## Test Types

### 1. Shell Interaction Tests (shell_interaction)
Used to test functions that require real interaction with command-line programs, such as:
- Program startup and basic operation
- Device type identification
- Text and image transmission
- Interactive console
- Stage-wise configuration support

### 2. Unit Tests (unit_test)
Used to test specific code modules and class implementations, such as:
- Frame structure and encoding/decoding classes
- CRC-16 checksum algorithm
- Switch and router functional modules
- Network protocol implementation

### 3. File Comparison Tests (file_comparison)
Used to verify output files generated by the program, such as:
- Automatic image file saving
- File naming format verification

## Usage Methods

### Running Unit Tests
```bash
# Run all unit tests
pytest evaluation/tests/

# Run a specific test file
pytest evaluation/tests/test_frame_structure.py

# Run a specific test function
pytest evaluation/tests/test_crc16.py::test_crc16_parameters
```

### Running Shell Interaction Tests
```bash
# Basic startup test
python code/main.py 2

# Test with input file
python code/main.py 2 < evaluation/port_query.in
```

### Batch Testing
```bash
# Windows batch file testing
bin\OneTouchToGo-stage2.bat
```

## Scoring Criteria

Each test point has a corresponding scoring standard:
- **2 points**: Function fully implemented and working correctly
- **1 point**: Function basically implemented but with minor issues
- **0 points**: Function not implemented or serious errors

A total of 30 test points, with weights ranging from 1 to 5, for a total weight of 75 points.

## Notes

1. Please ensure the system environment is configured correctly before running tests
2. Some tests require specific configuration files to exist
3. Network-related tests may require multiple terminal windows
4. It is recommended to execute tests in the order specified in the test plan
5. If a test fails, please check the relevant dependencies and prerequisites
