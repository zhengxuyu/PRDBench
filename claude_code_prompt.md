你需要为PRDBench评估框架开发一个**智能餐厅供应链分析与优化系统**。这是一个命令行交互的Python项目，用于评估代码agent的开发能力。

## 核心要求

### 1. 功能实现要求

基于 `PRDBench/1/src/PRD.md` 实现以下模块：

#### **2.1 菜品数据管理模块**
- ✅ 支持CRUD操作（增删改查）
- ✅ CSV批量导入功能
- ✅ 多条件搜索（ID、名称、分类）
- ✅ 删除确认机制

#### **2.2 成分构成分析模块**
- ✅ 成分清单管理（成分名称、用量、单位、成本单价）
- ✅ 成本结构分析（计算成分成本比例和毛利率，精确到小数点后1位）
- ✅ 过敏原识别（标记8大过敏原类别）

#### **2.3 销售数据分析模块**
- ✅ 订单数据导入（菜品ID、销售数量、销售时间、结算价格）
- ✅ 销售趋势分析（按日/周/月维度统计销售量变化）

#### **2.4 菜品相似度匹配模块**
- ✅ 基于名称的相似度算法
- ✅ 近似商品源文件上传，自动识别和归类相似菜品群组
- ✅ 统计分析（累积订单量、平均结算价、销售波动系数）

#### **2.5 命令行交互功能**
- ✅ 主菜单导航系统，支持模块间无缝切换
- ✅ 数据导入/导出进度可视化（文本进度条）
- ✅ 文本表格或简单ASCII图表显示分析结果
- ✅ 关键操作确认机制和错误处理

### 2. 技术实现要求

## 关键注意事项

### 🚨 CLI交互格式要求
```
主菜单：请选择功能模块
1. 菜品数据管理
2. 成分构成分析  
3. 销售数据分析
4. 菜品相似度匹配
5. 退出
请选择：
```

### 🚨 输出格式要求
- **成功消息**: "成功添加菜品 'Test Dish A'" (具体格式)
- **搜索结果**: 必须显示 "Record entry for Test Dish A"
- **错误处理**: 无数据时显示 "No data found."
- **确认操作**: 删除时必须要求用户确认（y/n）

### 🚨 文件结构要求
```
src/
├── main.py              # 程序入口，必须能用 python main.py 启动
├── dish_manager.py      # 菜品管理模块
├── ingredient_analyzer.py  # 成分分析模块
├── sales_analyzer.py    # 销售分析模块
├── similarity_matcher.py   # 相似度匹配模块
├── utils.py             # 工具函数
├── tests/               # 单元测试目录
│   ├── test_dish_manager.py
│   ├── test_ingredient_analyzer.py
│   └── test_sales_analyzer.py
└── README.md            # 项目说明文档
```

### 🚨 数据文件格式
- **菜品数据**: CSV格式，包含 菜品名称、分类、售价、烹饪时间
- **成分数据**: CSV格式，包含 成分名称、用量、单位、成本单价
- **订单数据**: 包含 菜品ID、销售数量、销售时间、结算价格

### 🚨 计算精度要求
- **毛利率**: 精确到小数点后1位 (例: 66.7%)
- **成本计算**: 准确计算总成分成本
- **销售统计**: 按时间维度正确聚合数据
